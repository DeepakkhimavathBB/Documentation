Executive Summary

This document describes the OffsetService, a specialized Azure Function responsible for reversing financial transactions. The service is triggered by an event containing a pointer to a transaction that needs to be offset. Its primary purpose is to create a set of new, opposing transactions that effectively cancel out the original transaction and all its related accounting entries across multiple database tables and schemas (dbo, adj).

The service systematically identifies the original transaction, its associated tax lots (TAL), tax lot offsets (TLO), and any corresponding adjustment (ADJ) entries. It then generates reversal records for each of these and packages them into a new payload for downstream persistence.

Goals and Scope

Primary Goals

  • To perform a complete and accurate reversal of a given financial transaction.
  • To create offsetting and reversal entries for associated TAL and TLO records.
  • To reverse corresponding entries in the adj schema.
  • To mark the original transaction records as reversed.
  • To produce a new transaction payload containing all the generated reversal records.

Scope

  • This document covers the internal architecture and reversal logic of the OffsetService.
  • It details the step-by-step process of how a transaction is reversed across different data entities.

High-Level Architecture

  • Trigger: Azure Event Grid (CloudEvent schema). The incoming event payload is expected to contain the original_agt_row_id of the transaction to be reversed.
  • Function App: OffsetService (Azure Function).
  • Entry Point: AtlasFunction.cs — Standard entry point that delegates processing to ProcessTask.
  • Orchestrator: Tasks/ProcessTask.cs — Core orchestration layer coordinating the full reversal workflow.
  • Processors: The orchestrator invokes multiple specialized processors:
    • ProcessRulesAndCreateJsonForAGT — Applies business rules to generate reversed AGT records.
    • TalReversalProcessor — Generates offsetting and reversal TAL records.
    • ProcessTloRecords — Generates offsetting and reversal TLO records.
    • ProcessAdjRecords — Reverses records in the adj schema.
  • Data Access: Uses DatabaseResultProcessor for data retrieval and IDatabaseTask for persistence.
  • Persistence: Produces a new blob containing reversal transactions and enqueues a new CloudEvent for downstream processing.

End-to-End Flow

  1. An Event Grid message triggers the AtlasFunction, providing the original_agt_row_id of the transaction to be reversed.
  2. The function invokes ProcessTask.ExecuteAsync().
  3. ProcessTask validates that original_agt_row_id is present.
  4. The service performs the reversal in a sequence of processing blocks:
    1. Main Transaction Reversal (dbo schema)
      1. Fetches the original agt_adv_glbl_trans record.
      2. Updates the original AGT record to mark it as reversed (SetReversalFlagAgt).
      3. Uses ProcessRulesAndCreateJsonForAGT to generate a new opposing AGT record.
      4. Fetches associated tal_tax_acct_ldgr (TAL) records.
      5. Invokes TalReversalProcessor to create and persist offsetting and reversal TAL records for each original tax lot.
    2. TLO Adjustment Reversal
      1. Searches for a related TLO Adjustment AGT record.
      2. If found, marks the original TLO adjustment AGT as reversed and generates a new opposing TLO adjustment AGT record.
      3. Fetches associated tlo_taxlot_offset (TLO) records.
      4. Invokes ProcessTloRecords to create and persist offsetting and reversal TLO records.
    3. Adjustment Schema Reversal (adj schema)
      1. Invokes ProcessAdjRecords.ProcessAdjReversalsAsync to reverse corresponding records in:
        • adj.agt
        • adj.gjs
        • adj.ged
  5. All newly generated reversed AGT records (from steps 4.a.iii and 4.b.ii) are collected into a TransactionRoot object.
  6. The TransactionRoot is serialized into a JSON payload.
  7. A new blob is created in storage containing the reversal transaction payload.
  8. A new CloudEvent referencing the new blob is enqueued for the downstream EvaluateService.

Component Responsibilities and Key Files

OffsetService.csproj

Defines project dependencies and embedded SQL files (e.g., GetAgt.sql, UpdateAgt.sql).

AtlasFunction.cs

Azure Function entry point delegating execution to ProcessTask.

Program.cs

Configures dependency injection and registers all processors (TalReversalProcessor, ProcessTloRecords, ProcessAdjRecords).

Tasks/ProcessTask.cs

Main orchestrator coordinating the multi-stage reversal process.

Tasks/DatabaseResultProcessor.cs

Handles database fetch operations using embedded SQL.

Tasks/TalReversalProcessor.cs

Implements offsetting and reversal logic for TAL records.

Tasks/ProcessTloRecords.cs

Implements offsetting and reversal logic for TLO records.

Tasks/ProcessAdjRecords.cs

Implements reversal logic for adj schema records.

Tasks/ProcessRulesAndCreateJsonForAGT.cs

Applies rules to generate reversed AGT transaction objects.

Error Handling and Resiliency

  • The execution of ProcessTask.ExecuteAsync is wrapped by RunProcessTaskAsync, provided by the AtlasExecutableTask base class.
  • This wrapper enforces standardized logging, exception handling, and dead-letter behavior.
  • A NonRecoverableException is explicitly thrown if the provided original_agt_row_id cannot be found, preventing retries for invalid requests.
  • The service integrates with IIdempotencyService to ensure that duplicate reversal events do not result in duplicate processing.
